## 📋 完整产品需求文档

### 1. 产品概述
一款将音频文件实时转化为动态视觉艺术的Web应用。用户上传音乐后，系统通过音频分析技术，生成与音乐节奏、旋律、情感高度契合的唯美动态画面，打造沉浸式的视听体验。

### 2. 目标用户
- 音乐爱好者（想以新方式感受音乐）
- 内容创作者（制作音乐可视化视频）
- 冥想/疗愈需求人群（治愈系画面）
- 派对/活动现场（酷炫背景墙）

### 3. 核心功能需求

#### 3.1 音乐上传与管理
- **格式支持**：MP3, WAV,（主流音频格式）
- **上传方式**：拖拽上传 + 点击选择
- **文件限制**：≤50MB（平衡用户体验与服务器压力）
- **预览功能**：上传后显示文件名、时长、波形预览图
- **示例音乐**：提供3-5首内置示例（方便用户快速体验）

#### 3.2 可视化模式（核心）
**至少提供3种风格模板供选择：**

| 风格 | 视觉特点 | 适用场景 |
|------|----------|----------|
| **唯美·星河流转** | 粒子系统构成星云，随音乐缓慢旋转，高频时闪烁 | 抒情、古典、民谣 |
| **酷炫·能量脉冲** | 几何图形+光束，节奏感强烈的跳动、扩散 | 电子、摇滚、流行 |
| **治愈·水彩晕染** | 柔和色彩渐变，如水墨般扩散、交融 | 轻音乐、冥想、钢琴 |

**进阶功能**：智能匹配（AI根据音乐情感自动推荐最合适的风格）

#### 3.3 节奏同步机制
- **FFT频谱分析**：实时解析音频的低、中、高频
- **节拍检测**：识别鼓点、重拍，触发视觉高光时刻（如闪光、爆炸效果）
- **音量映射**：音量大小影响粒子大小/运动速度/色彩饱和度
- **旋律平滑度**：平滑段落对应缓慢流动，高潮对应剧烈变化

#### 3.4 用户控制面板
- **播放控制**：播放/暂停、进度条拖动、音量调节
- **视觉微调**：色彩主题（可切换预设色板）、粒子密度、运动强度
- **全屏模式**：沉浸式体验
- **录制/导出**：录制30秒-1分钟的视频（MP4格式），方便分享（进阶功能）

#### 3.5 辅助功能
- **响应式设计**：完美适配桌面端（Chrome优先）+ 移动端基础适配
- **加载动画**：上传/分析过程中的等待状态
- **错误提示**：格式不支持、文件过大等情况

### 4. 技术实现要点

#### 4.1 前端技术栈
- **框架**：React + Vite / Vue3（快速开发）
- **可视化库**：
  - **Three.js**：3D粒子、几何图形渲染（适合酷炫效果）
  - **p5.js / Canvas API**：2D唯美、治愈风格
  - **D3.js**：波形绘制
- **音频处理**：**Web Audio API**（浏览器原生，无需后端）
  - `AnalyserNode` 获取频谱数据
  - `AudioContext` 解码音频

#### 4.2 核心算法逻辑
```javascript
// 伪代码示例
1. 用户上传 -> AudioContext.decodeAudioData()
2. 创建AnalyserNode，连接音频源
3. requestAnimationFrame循环：
   - 获取频域数据（getByteFrequencyData）
   - 根据低频（ bass）控制背景颜色/粒子大小
   - 根据中频（ mid）控制粒子运动速度
   - 根据高频（ treble）控制闪烁/透明度
   - 检测能量峰值（beat detection）触发特效
4. 更新Three.js场景或Canvas绘图
```

#### 4.3 性能优化
- 使用OffscreenCanvas（如果复杂度高）
- 限制粒子数量（≤5000）保证60fps
- 移动端自动降低特效质量

#### 4.4 部署方案
- **静态托管**：Vercel / Netlify / GitHub Pages
- **域名**：可选（如 audiodream.xyz）

```
开始 → 上传音乐 → 预览波形 → 选择风格（或智能匹配）

```
